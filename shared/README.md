# shared/

Things the public **website** and the **admin** dashboard both need, kept in one place so they don't drift apart.

## Contents

- **`design-tokens/tokens.css`** — the canonical color, type-scale, and spacing values (black & gold system). This is the single source of truth.
  - `website/assets/css/styles.css` keeps its own copy of the same `:root` values, because the website folder must stay self-contained and independently deployable (see `docs/DEPLOYMENT.md`). It is **not** allowed to silently drift — run `check-sync.py` after changing either file.
  - The admin dashboard should import/reference this file directly (or read the same hex values from it) rather than hard-coding its own copy.
  - `check-sync.py` — run this after editing colors/spacing anywhere: `python3 shared/design-tokens/check-sync.py`. It exits non-zero if the website's copy has drifted from the canonical file.

- **`icons/icon-set.json`** — the 17 inline SVG icons used across the website (shield, factory, hospital, clock, check, arrow, etc.), extracted from `website/build.py` into one JSON registry: `{ "I_SHIELD": "<svg>...</svg>", ... }`. Use this so the admin dashboard's icon pickers (e.g. when choosing an icon for a new Service or Industry entry) show the exact same icon set as the live site, instead of a different icon library.

## Why this exists

Version 1 of the website (plain HTML/CSS/JS, generated by `build.py`) and Version 1 of the admin dashboard (planned next) are built with different tech. Rather than trying to force them to share a build pipeline right now, this folder holds the handful of things — brand colors and icons — that genuinely must stay identical between the two, with a script to catch drift instead of a promise to remember.

## Version 2 note

If the site is ever rebuilt with a shared component framework (e.g. both website and admin built in React/Next.js), this folder is the natural place for real shared UI components too (`shared/components/Button.jsx`, `Card.jsx`, etc.) instead of just tokens and icons.
